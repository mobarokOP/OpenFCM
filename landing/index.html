Developer-first

Ship it in an afternoon

One dependency on the device, one HTTP call from your backend. That's the whole surface area.

build.gradle.kts · App.kt
// One dependency — no google-services.json, no Firebase SDK
dependencies {
    implementation("com.github.mobarokOP:OneFCM:1.1.0")
}

// Application.onCreate() — the whole integration
OpenFCM.init(context, appId = "YOUR_APP_ID")

// Optional: identify & target this device
OpenFCM.login("user_42")
OpenFCM.setTags(mapOf("plan" to "pro", "lang" to "en"))
OpenFCM.subscribeTopic("release-notes")
send-notification.sh
# Send to every "pro" user — from any backend
curl -X POST https://api.onefcm.com/v1/notifications \
  -H "Authorization: Bearer $ONEFCM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "app_id":   "YOUR_APP_ID",
    "title":    "Flash sale ⚡ 40% off Pro",
    "body":     "Ends in 2 hours — tap to claim",
    "url":      "myapp://sale",
    "audience": { "tags": { "plan": "pro" } }
  }'
How it works

Live in four steps

From zero to a push on a real device in under ten minutes.

1

Create an app

In the dashboard, add your app with its package name and copy the App ID.

2

Upload one file

Drop your Firebase service-account.json in app settings. Everything else — keys, project ID, isolation — is derived automatically.

3

Add the SDK

One Gradle line plus OpenFCM.init(context, APP_ID). Devices register themselves and permission is prompted for you.

4

Send

Compose in the dashboard or POST to /v1/notifications from your server — then watch sent, delivered and opened tick up live.

FAQ

Questions, answered

The things every Android team asks before switching push providers.

Is OneFCM free and open-source?
The Android SDK is open source on GitHub, and the platform is built to run on your own infrastructure — so there's no per-subscriber toll on your growth.
Do I need google-services.json in my app?
No. That's the point. Your Firebase credentials live on the server as a single service-account upload; the SDK fetches everything it needs from your OneFCM instance at runtime. Your APK ships with zero Firebase configuration.
Does it work when the app is closed?
Yes — notifications are delivered by Firebase Cloud Messaging at the OS level, so they arrive when the app is backgrounded or killed. The usual Android caveat applies: a few OEM battery savers (some Xiaomi, Huawei and Oppo builds) can delay pushes for apps the user has force-stopped.
Can I self-host it?
Absolutely — that's the default. Run the platform on your own VPS, point the SDK's baseUrl at your domain, and every token, tag and delivery log stays in your database. Nothing transits a third party except FCM itself.
How do deep links work?
Attach a url or custom data to any notification. On tap, the SDK routes app links (like myapp://sale) to the right screen, opens web URLs in the browser, and reports the click back for CTR analytics — no extra code in your activities.
How is my data kept isolated?
Every app you create pushes through its own Firebase project via its own service account — no shared sender, no cross-app leakage. API keys are stored hashed, service-account credentials are encrypted at rest, and delivery logs are scoped per app.
